# Web UI for eval-driven-dev artifects

The goal is to create a web UI that renders local artifacts generated by eval-driven-dev skills.

## Generated Artifacts

Currently the skill generates the following artifacts in the configured pixie-qa folder (default to pixie_qa/) that need to be rendered in the web UI:

- markdown files: entry-point, data-flow, eval-criteria etc
- datasets
- scorecards

Attached an pixie_qa folder as example.

## How the web UI should work

- a `pixie start` command should open the web UI in web browser, if it's not already open.

- the web UI should have tabs on top: scorecards, datasets, then one tab per md file.

- select tab to show the content of related artifact(s).

- for md, simply format the content to HTML.

- for scorecards and datasets, they should have a list on the left for selecting specific scorecard/dataset to view. On the right it should be the main panel to render selected dataset of scorecard.

- scorecard content should be shown with iframe.

- dataset content should be rendered in a table.

- the web UI should be updated live when any of the artifact(s) change, including addition/removal. The tab/list selection should also switch to the updated artifact, when update happen.

- the header currently in scorecard should be added to the new web UI; and when scorecard is rendered in the iframe it should hide the header.

## technical requirements

- the web UI should be implemented in the existing frontend codebase and in react + ts, and got built into the pixie-qa's assets, similar to how the scorecard is currently build & deployed.

- the web UI should be served by a python http server, and updates should be pushed to the frontend with SSE

- implement the http server with starlette + uvicorn.

- use watchfiles to watch for artifact addition/removal/update, and trigger SSE accordingly

- the `pixie start` command should start the http server if no server is running, and open the frontend in webbrowser if no client is subscribed to the server's SSE currently.
