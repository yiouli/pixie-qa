# Pixie Frontend

React single-page applications for the pixie-qa package. Contains two build targets:

1. **Scorecard** — self-contained HTML report generated by `pixie test`
2. **Web UI** — live-updating dashboard served by `pixie start`

Both are compiled to single HTML files that ship inside the Python package.

## Quick Start

```bash
# Install dependencies
npm install

# Start scorecard dev server with mock data
npm run dev

# Start web UI dev server
npm run dev:webui

# Build both for production (outputs to ../pixie/assets/)
npm run build

npm run build:webui        # → ../pixie/assets/webui.html
```

## Stack

- **React 19** + **TypeScript 5.7**
- **Vite 6** bundler
- **vite-plugin-singlefile** — inlines all JS/CSS into one HTML file

## Build System

The build uses `VITE_BUILD_TARGET` env var to select which entry point to compile:

- `VITE_BUILD_TARGET=scorecard` (default) → builds `index.html` → `../pixie/assets/index.html`
- `VITE_BUILD_TARGET=webui` → builds `webui.html` → `../pixie/assets/webui.html`

`npm run build` runs both targets sequentially. Each produces a single self-contained HTML file with all JS/CSS inlined.

## How It Works

### Scorecard (inside `pixie test`)

1. Python's `generate_scorecard_html()` loads the compiled `index.html` template
2. Replaces `"__PIXIE_DATA_PLACEHOLDER__"` with JSON.dumps(report_data)
3. The saved HTML file opens standalone (`file://`) — no server needed, no external deps
4. When embedded in the web UI iframe, the BrandHeader is automatically hidden

### Web UI (inside `pixie start`)

1. Python starts a Starlette server that serves `webui.html` and API endpoints
2. The React app fetches `/api/manifest` for the list of artifacts
3. Connects to `/api/events` (SSE) for live change notifications
4. Scorecards are displayed in iframes; datasets as tables; markdown files as formatted HTML
5. File changes trigger automatic tab/selection updates

### Development

```bash
npm run dev          # Scorecard dev server at http://localhost:5173
npm run dev:webui    # Web UI dev server
```

In dev mode, `main.tsx` fetches `/mock-data.json` instead of reading
`window.PIXIE_REPORT_DATA`. Edit `public/mock-data.json` to test different data shapes.

### Build

```bash
npm run build
```

Produces two files in `../pixie/assets/`:

- `index.html` — scorecard (~214 KB, ~66 KB gzip)
- `webui.html` — web UI (~211 KB)

Both are single HTML files with all React code, CSS, and assets inlined. These files are gitignored because they are build artifacts; they must be rebuilt after any frontend changes.

## Data Interface

The React app expects a `ScorecardReportData` object (see `src/types.ts`):

```typescript
interface ScorecardReportData {
  command_args: string;
  timestamp: string;
  summary: string; // e.g. "2/4 tests passed"
  pixie_repo_url: string;
  feedback_url: string;
  brand_icon_url: string;
  test_records: TestRecordData[];
}
```

Each `TestRecordData` contains `name`, `status`, optional `message`, and an array
of `AssertRecordData` with the full evaluation tensor (`results[passes][inputs][evaluators]`).
